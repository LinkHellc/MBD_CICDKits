# Product Brief: MBD_CICDKits

## Executive Summary

**MBD_CICDKits** 是一款面向嵌入式开发工程师的桌面自动化工具，专注于 Simulink 模型开发的 CI/CD 流程自动化。

### 核心价值主张

| 维度 | 改进 |
|------|------|
| ⏱️ 效率提升 | 60 分钟 → 15 分钟 |
| 🎯 可靠性 | 98% 成功率 |
| 🔧 可操作性 | 清晰的错误诊断与修复建议 |
| 👥 团队协作 | 配置通过 Git 共享 |

### 成功愿景

- **3 个月**：3-5 名工程师主动试用，有人主动分享给同事
- **6 个月**：60% 以上工程师日常使用，平均每次节省 30-40 分钟
- **12 个月**：成为新员工入职培训内容，手动操作错误减少 90%

**关键成功信号**："这工具成功了" = 有人主动分享给同事说"试试这个工具"

## Core Vision

### Problem Statement

当前热管理 ECU 开发流程存在以下痛点：

| 痛点 | 描述 |
|------|------|
| **低效率** | 手动执行 MATLAB 代码生成、文件处理、IAR 编译、A2L 处理，耗时 45-60 分钟 |
| **易出错** | 复杂的手动步骤（排除特定文件、编辑 Cal.c）容易遗漏或出错 |
| **难学习** | 新员工需要 2 天培训才能掌握完整流程 |
| **难排查** | 构建失败时难以定位问题 |

### Problem Impact

- **工程师时间浪费**：每次构建消耗 45-60 分钟宝贵开发时间
- **培训成本高**：新员工需要 2 天才能独立完成构建
- **质量风险**：手动操作容易出错，影响交付质量
- **团队效率**：重复性劳动占用创新时间

### Why Existing Solutions Fall Short

现有方案存在的不足：

1. **手动流程**：依赖二十多页操作文档，步骤繁琐易错
2. **脚本工具**：缺乏 UI 界面，配置复杂，难以推广
3. **商业 CI/CD**：不支持 Simulink 模型开发特定需求（A2L 处理、IAR 集成）

### Proposed Solution

MBD_CICDKits 提供 5 阶段完整自动化构建流程：

```
1. 项目配置阶段 → 2. MATLAB代码集成 → 3. IAR工程编译 → 4. A2L文件处理 → 5. 最终文件归纳
```

**核心特性**：
- 📋 可配置工作流支持不同使用场景
- 📊 实时进度显示和阶段状态追踪
- 🔍 结构化日志和可操作的错误提示
- 🔄 配置保存与快速加载
- 👥 团队配置共享（通过 Git）

### Key Differentiators

| 差异点 | 说明 |
|--------|------|
| **垂直领域专注** | 针对 Simulink 模型开发的 CI/CD，非通用工具 |
| **可操作的错误诊断** | 不止报错，还告诉你是第几步出错、原因是什么、如何修复 |
| **新员工友好** | 通过工具界面"看懂"流程，降低学习曲线 |
| **团队协作设计** | 配置即代码，Git 共享，统一团队标准 |
| **可靠性优先** | 98% 成功率目标，错误可恢复（断点续跑） |
